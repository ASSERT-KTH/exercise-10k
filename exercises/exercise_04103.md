## Exercise
Write a program that demonstrates the use of valid and invalid variable names, and explains the rules for naming variables in the chosen programming language.

## Reference Solution
```python
# Valid variable names
a = 10
b = 20
c = 30

print(a)
print(b)
print(c)

# Invalid variable names
# 1a = 10  # SyntaxError: invalid syntax
# a! = 20  # SyntaxError: invalid syntax
# a b = 30  # SyntaxError: invalid syntax

# Valid variable names with special characters
_a = 10
_a_b = 20
_a1 = 30

print(_a)
print(_a_b)
print(_a1)

# Rules for naming variables in Python:
# 1. Variable names can only contain letters (a-z or A-Z), digits (0-9), and underscores (_).
# 2. Variable names cannot start with a digit.
# 3. Variable names are case-sensitive.
# 4. Variable names cannot be the same as Python keywords.

# For example, the following variable names are invalid:
# 1a, a!, a b, class, def, for, while, etc.

# Best practices for naming variables in Python:
# 1. Use descriptive and meaningful variable names.
# 2. Use lowercase letters with words separated by underscores.
# 3. Avoid using special characters other than underscores.
# 4. Avoid using very long or very short variable names.
```
