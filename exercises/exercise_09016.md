## Exercise
Write a function that demonstrates error handling using try-except blocks.

## Reference Solution
```python
def divide_numbers(num1, num2):
    """
    Divide two numbers and handle potential errors.

    Args:
        num1 (float): The dividend.
        num2 (float): The divisor.

    Returns:
        float: The result of the division.

    Raises:
        TypeError: If either num1 or num2 is not a number.
        ZeroDivisionError: If num2 is zero.
    """
    try:
        # Attempt to divide the numbers
        result = num1 / num2
        return result
    except TypeError:
        # Handle the case where either num1 or num2 is not a number
        print("Error: Both inputs must be numbers.")
    except ZeroDivisionError:
        # Handle the case where num2 is zero
        print("Error: Cannot divide by zero.")
    except Exception as e:
        # Handle any other unexpected errors
        print(f"An unexpected error occurred: {str(e)}")

# Example usage
if __name__ == "__main__":
    # Test case 1: Valid division
    print("Test case 1:")
    print(divide_numbers(10, 2))

    # Test case 2: Division by zero
    print("\nTest case 2:")
    divide_numbers(10, 0)

    # Test case 3: Invalid input type
    print("\nTest case 3:")
    divide_numbers(10, "a")
```
